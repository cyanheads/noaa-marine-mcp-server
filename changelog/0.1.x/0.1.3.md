# 0.1.3 — 2026-06-06

## Changed

- **`noaa_marine_find_stations`** — description notes that CO-OPS current stations are cataloged by monitoring capability, not prediction availability; directs agents to try the next nearest station when `noaa_marine_get_currents` returns `no_predictions` ([#2](https://github.com/cyanheads/noaa-marine-mcp-server/issues/2)).
- **`noaa_marine_get_currents` direction** — 6-min prediction `direction` field is now `number | null`; absent `Direction` in CO-OPS response maps to `null` instead of silently defaulting to `0°` ([#4](https://github.com/cyanheads/noaa-marine-mcp-server/issues/4)).

## Fixed

- **CO-OPS body-level errors** — `checkCoopsError()` now maps known error messages to a typed `CoopsBodyError` (extends `McpError`, `retryable: false`) with a `coopsReason` discriminator (`no_predictions`, `station_error`, `no_data`); tools map these to typed contract reasons (`ctx.fail(...)`) instead of generic `ServiceUnavailable` with retry ([#1](https://github.com/cyanheads/noaa-marine-mcp-server/issues/1)).
- **Tide predictions `interval=6min`** — `fetchTidePredictions` now translates `interval='6min'` to `interval='6'` before sending to the CO-OPS API, matching what `fetchCurrentPredictions` already did; previously caused "No Predictions data was found" for every 6-min tide request ([#3](https://github.com/cyanheads/noaa-marine-mcp-server/issues/3)).
