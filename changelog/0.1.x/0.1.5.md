# 0.1.5 — 2026-06-20

## Added

- **`check-dependency-specifiers` devcheck step** — new `scripts/check-dependency-specifiers.ts`, registered in `devcheck` (`Dependency Specifiers`, flag `--no-dep-specifiers`), hard-fails on floating specifiers (`latest`, `*`, pre-release dist-tags) in `package.json`'s dependency sections and `bun.lock`'s `workspaces` map. Static and local — runs in the default and `--fast` passes. ([cyanheads/mcp-ts-core#246](https://github.com/cyanheads/mcp-ts-core/issues/246))
- **Plugin-manifest packaging checks** — `lint:packaging` validates `.claude-plugin/plugin.json` and `.codex-plugin/plugin.json`: non-empty descriptions, unscoped machine name in display fields, full scoped name in the `npx -y` install arg. Gated by the new `devcheck.config.json` `packaging.pluginManifests` flag (enabled here); the `devcheck` Packaging gate now runs when a plugin manifest is present. ([cyanheads/mcp-ts-core#240](https://github.com/cyanheads/mcp-ts-core/issues/240))
- **`ctx.content` collector** — `@cyanheads/mcp-ts-core` ^0.10.8 adds an always-present `Context` collector for non-text content blocks (`ctx.content.image()` / `.audio()`); collected blocks prepend to `content[]` and never enter `structuredContent`. Available to handlers; not yet used by this server. ([cyanheads/mcp-ts-core#239](https://github.com/cyanheads/mcp-ts-core/issues/239))

## Changed

- **Fresh-scaffold devcheck guards** — synced from mcp-ts-core 0.10.7: `build-changelog.ts` exits cleanly under `--check` when no per-version files exist, `check-framework-antipatterns.ts` self-guards outside a git repo, `check-skill-versions.ts` skips a `SKILL.md` deleted from the worktree, and `devcheck.ts` skips git-dependent checks when `.git` is absent. ([cyanheads/mcp-ts-core#242](https://github.com/cyanheads/mcp-ts-core/issues/242), [#243](https://github.com/cyanheads/mcp-ts-core/issues/243), [#237](https://github.com/cyanheads/mcp-ts-core/issues/237))
