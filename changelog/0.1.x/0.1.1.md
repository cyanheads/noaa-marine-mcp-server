# 0.1.1 — 2026-06-05

Initial public release of `@cyanheads/noaa-marine-mcp-server`. Provides 5 MCP tools and 1 resource over the [NOAA CO-OPS Tides & Currents API](https://api.tidesandcurrents.noaa.gov/api/prod/) and [NDBC](https://www.ndbc.noaa.gov/) buoy data:

- **`noaa_marine_find_stations`** — discover CO-OPS tide/water-level/current stations and NDBC buoys by location (lat/lon proximity) or name/state; returns unified station list with source, type, capabilities, and coordinates
- **`noaa_marine_get_tide_predictions`** — high/low tide predictions for a CO-OPS tide station over a date range (hilo or 6-min intervals, configurable datum)
- **`noaa_marine_get_water_level`** — observed water level with paired predictions for residual (storm surge / anomalous drawdown) computation; 6-min resolution, up to 31 days per request
- **`noaa_marine_get_currents`** — tidal current predictions for a CO-OPS current station (MAX_SLACK or 6-min continuous); flood/ebb speeds, slack times, directions
- **`noaa_marine_get_conditions`** — live marine conditions from an NDBC buoy: wave height/period/direction, wind speed/gust/direction, SST, air temperature, barometric pressure, dew point
- **`noaa-marine://station/{station_id}`** resource — metadata for a CO-OPS or NDBC station by ID: name, coordinates, source, capabilities, and state

## Added

- **`noaa_marine_find_stations`**, **`noaa_marine_get_tide_predictions`**, **`noaa_marine_get_water_level`**, **`noaa_marine_get_currents`**, **`noaa_marine_get_conditions`** — full 5-tool surface over CO-OPS and NDBC
- **`noaa-marine://station/{station_id}`** resource — unified station metadata across CO-OPS and NDBC sources
- **`CoopsService`** — CO-OPS JSON API client with station cache and parallel station lookups
- **`NdbcService`** — NDBC active-stations XML parser and realtime observation text reader
- **STDIO and Streamable HTTP** transport support; optional `NOAA_APPLICATION_ID` env var for CO-OPS courtesy identification

## Security

- **Station ID regex** — added `.regex(/^[A-Za-z0-9_-]{1,20}$/)` to all `station_id` inputs (5 tools + 1 resource) and stripped raw upstream URL from error messages to prevent log injection
