# 0.1.4 — 2026-06-12

## Added

- **`noaa_marine_find_stations` `truncated`** — optional output boolean, set when `total_found` exceeds the requested `limit`. The handler also emits `ctx.enrich.truncated(...)` and the text header flags truncated results so agents know to raise the limit or narrow filters.
- **Explicit server identity** — `createApp()` now sets `name` and `title` to `noaa-marine-mcp-server` rather than relying on the npm-scoped package name.

## Changed

- **CO-OPS error codes** — `station_not_found` now maps to `JsonRpcErrorCode.NotFound` (was `InvalidParams`) and `date_range_exceeded` to `JsonRpcErrorCode.ValidationError` (was `InvalidParams`) across `noaa_marine_get_tide_predictions`, `noaa_marine_get_water_level`, and `noaa_marine_get_currents`. Reason discriminators are unchanged.
- **`bundle` script** — runs `scripts/clean-mcpb.ts` after `mcpb pack`: `mcpb clean` prunes dev dependencies, then dependency-shipped agent docs nested under `node_modules/` (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) are stripped from the `.mcpb` — root-anchored `.mcpbignore` patterns cannot reach them. `lint:packaging` gains a post-bundle content check that fails if any such entry survives.
- **`Dockerfile`** — adds an `APP_VERSION` build arg wired to the `org.opencontainers.image.version` OCI label and a `HEALTHCHECK` against `/healthz` in HTTP mode.
- **`.mcpbignore`** — patterns anchored with a leading `/` so they exclude only repo-root entries, not same-named paths nested in dependencies.
- **`check-framework-antipatterns.ts`** — adds rule 4 flagging `z.coerce.boolean()` on env flags (use `z.stringbool()`); comment lines are skipped so documenting the pattern doesn't trip the scan.
