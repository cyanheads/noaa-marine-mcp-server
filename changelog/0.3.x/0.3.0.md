# 0.3.0 — 2026-07-16

## Added

- **`noaa_marine_get_ocean_observations`** — new tool reading an NDBC station's `.ocean` realtime feed for the latest sub-surface water-column observation: at each reported depth, water temperature (°C), conductivity (mS/cm), salinity (psu), dissolved oxygen (% saturation and ppm), chlorophyll (µg/l), turbidity (FTU), pH, and redox potential (mV). A station reporting several depths at one timestamp returns one reading per depth. The water-column counterpart to `noaa_marine_get_conditions`, which returns surface meteorology and sea state. Ocean-sensor coverage carries no station-catalog flag, so it is a best-effort call on NDBC station IDs — a typed `observations_not_found` error is returned where no `.ocean` file exists, and `latitude`/`longitude` are `null` when the station is absent from the active-stations list. ([#16](https://github.com/cyanheads/noaa-marine-mcp-server/issues/16))
