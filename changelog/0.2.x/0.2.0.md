# 0.2.0 — 2026-07-16

## Added

- **`noaa_marine_get_current_profile`** — new tool reading an NDBC station's `.adcp` realtime feed for the latest observed ocean-current profile: depth (m), direction (degrees true), and speed (cm/s) per depth bin. Distinct from `noaa_marine_get_currents`, which returns CO-OPS tidal-current *predictions*, not observed data. ([#15](https://github.com/cyanheads/noaa-marine-mcp-server/issues/15))
- **`current_profile`** — new `types` filter value on `noaa_marine_find_stations`, reaching NDBC stations with observed-current data; previously no filter value matched them. ([#15](https://github.com/cyanheads/noaa-marine-mcp-server/issues/15))

## Changed

- **`platform`** — new field on `noaa_marine_find_stations` NDBC rows and the station resource: the NDBC physical platform class (`buoy`, `fixed`, `oilrig`, `dart`, `tao`, `usv`, `other`). Absent for CO-OPS, which has no platform taxonomy. ([#14](https://github.com/cyanheads/noaa-marine-mcp-server/issues/14))
- **`type`** — now exclusively the data-capability axis (`tide`, `current`, `water_level`, `met`, `current_profile`), consistent between `find_stations` and the station resource, and optional — omitted when a station has no data capability. The resource previously emitted CO-OPS's undocumented station reference code (R/T/S) here instead. ([#14](https://github.com/cyanheads/noaa-marine-mcp-server/issues/14))
- **NDBC observed-currents capability** — renamed `currents` → `current_profile` in `capabilities`, `type`, and the `types` filter, removing its one-letter collision with CO-OPS's `current` (tidal-current predictions). ([#15](https://github.com/cyanheads/noaa-marine-mcp-server/issues/15))

## Fixed

- **Fabricated `buoy` capability** — `noaa_marine_find_stations` and the station resource no longer report `capabilities: ["buoy"]` / `type: "buoy"` for an NDBC station with no reported met or current data; such a station now reports an empty `capabilities` array, and the `types` filter's `buoy` value matches a station's real platform class instead of falling back to any station with no other capability. ([#13](https://github.com/cyanheads/noaa-marine-mcp-server/issues/13))
